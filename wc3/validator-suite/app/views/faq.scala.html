What is the W3C Validator Suite™?

The W3C Validator Suite™ service checks an entire Web site and evaluates conformance with W3C open standards. Validation currently includes HTML, CSS and Internationalization checking.

Why should I validate a Website?

Simply checking that a Website displays well in several browsers might prove that the site delivers a good experience today, but it doesn't guarantee that it will do so tomorrow. Validation is one of the simplest ways to check a site's conformity to Web standards now and for the future. Validation also may be used as a debugging tool - to test your HTML and CSS code and make sure your pages are served appropriately for international visitors. Validation simply is a good practice to adopt, and shows you are a true Web professional.

What is the difference between the existing free W3C Markup Validator and this new service?

W3C Validator Suite™ integrates Web standards conformance tools to make it easy and quick to keep your entire Web site compliant with the latest Web standards.

W3C will continue to offer its free validator tools, including the single page Markup Validator. This free service only checks one Web page at a time.

When I see the W3C Validator Suite™ logo/badge, what does it mean?

The service logo refers to the five key benchmarks for the Open Web Platform. The W3C Validator Suite™ badge says "Improved with W3C Validator Suite". You are encouraged to post the badge to your Websites to indicate you used the service, although it does not imply any seal of approval or service outcomes.

What about Accessibility testing?

Currently the W3C Validator Suite™ does not test for accessibility. Additional conformance checking for accessibility, mobile, etc. is under consideration for the near future.

How long will it take to validate my Web site?

Validation processing time depends on multiple factors like the number of your Web pages, the complexity of your page content, your server's capacity, and your network bandwidth. As an example, an average Web site of 900 pages could be processed in 15 to 20 minutes.

Can I check internal websites or sites that require authentication?

No, the W3C Validator Suite™ can only check sites that are reachable from the Internet without passwords.

My site uses Javascript. Is that OK?

Some users do not enable Javascript in their browsers. Therefore, to be fully interoperable your Web site should not depend on navigation links inside the Javascript. W3C Validator Suite™ will not process the Javascript; it will only follow the links that are specified in the HTML markup. If the Javascript modifies the HTML, W3C Validator Suite™ will only check the original, unmodifed HTML. Aside from this, Javascript in your site should not impact the W3C Validator Suite™ report.

Is the W3C Validator Suite™ published under an open-source license?

Yes, the W3C Validator Suite code is distributed under the W3C Software License and is availabe on GitHub. It is also based on existing W3C Validators which are themselves available under an open-source license.
